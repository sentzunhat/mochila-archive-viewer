## Problem

The frontend styling is scattered across `src/frontend/src/style.css` with:

1. **Hardcoded hex values** in several places (`#171717`, `#f6f0d5`, `#f1edc9`, `#fffdd9`, `#111`) that should reference CSS custom properties instead of the existing `:root` tokens
2. **No centralized design guide** — new contributors and AI agents have no single reference for color choices, spacing values, border radii, or typography scale
3. **Tailwind config drift** from `style.css` custom properties — Tailwind extends with `archive.bg`, `archive.panel`, etc., but these are not referenced in a unified style guide document
4. **Typography inconsistency** — body uses Inter (from CSS), while Tailwind defines serif as Iowan Old Style/Palatino and sans as Avenir Next/Segoe UI, with no documented rationale for when each applies
5. **No spacing scale** — values like `8px`, `10px`, `16px`, `18px`, `20px`, `34px` appear ad-hoc across layouts without a shared scale reference

## Evidence (style.css audit)

### Current CSS custom properties (foundation exists ✓)

```css
:root {
  --bg: #fbfaf2; /* warm paper */
  --ink: #181712; /* near-black text */
  --muted: #6f6b58; /* secondary text */
  --line: #ded7ad; /* borders, dividers */
  --panel: #fffef8; /* card surfaces */
  --snap: #fffc00; /* accent (Snapchat yellow) */
  --snap-dark: #cfc600; /* accent active/border */
}
```

### Hardcoded values that should use tokens ✗

| Line | Value                | Context                                   | Should be                               |
| ---- | -------------------- | ----------------------------------------- | --------------------------------------- |
| 265  | `#171717`            | `.preview` background (media tile dark)   | `--panel` or new `--surface-dark`       |
| 277  | `#f6f0d5`            | `.placeholder` text color                 | Derived from `--muted` with alpha       |
| 375  | `#fffef8`, `#f1edc9` | Linear gradient on `.bar span` or similar | `--panel`, new `--accent-light`         |
| 420  | `#fffdd9`            | Highlight/surface tint                    | `var(--bg)` with opacity or named token |
| 520  | `#111`               | Background fallback                       | `var(--ink)` or `rgba(0,0,0,0.87)`      |

### Missing token categories (no design guide exists)

- **Spacing scale** — should define: `4`, `8`, `12`, `16`, `24`, `32` as named tokens (e.g., `--space-xs`, `--space-sm`, `--space-md`, etc.)
- **Border radius scale** — currently only `8px` and `999px`; should define: `--radius-sm(4)`, `--radius-md(8)`, `--radius-lg(12)`, `--radius-full`
- **Typography scale** — h1 (`28px`), h2 (`18px`), body (`inherit`), captions/labels (`12px`–`13px`) — documented with weight/luminance tiers
- **Elevation/shadows** — none currently; header uses `backdrop-filter: blur(16px)` for glass effect (should be a token)

## Scope (design documentation only — no code changes in this item)

### Deliverable 1: `.hawp/kit/examples/design-system.md`

A concise design reference documenting:

1. **Color palette** — all 8 CSS vars with names, hex values, and usage rules
   - `bg` = page background (warm paper)
   - `ink` = primary text (near-black, high contrast)
   - `muted` = secondary/tertiary text, timestamps, labels
   - `line` = borders, dividers, tab separators
   - `panel` = card surfaces, list backgrounds
   - `snap` = accent fills (active tabs, badges)
   - `snap-dark` = accent borders, active state strokes
2. **Accessibility baseline** — confirm WCAG AA for ink on bg (`181712` on `fbfaf2` ≈ 14:1 ✓), muted on panel (`6f6b58` on `fffef8` ≈ 4.6:1 ✓)

3. **Spacing scale** — current values observed in style.css: `0, 4, 5, 8, 9, 10, 12, 13, 14, 16, 18, 20, 24, 34, 38`. Roughly 4px-based but with intentional irregularities for visual rhythm (e.g., `18px` padding on header/main, `34px` bottom main padding, `5px` label margin, `9px` year-list item padding). Proposed canonical scale should note which values to absorb and which are deliberate deviations.

4. **Border radius convention** — `--radius-sm(4)`, `--radius-md(8)`, `--radius-full(999)`

5. **Typography scale** — document current sizes, weights, and the Inter font stack with fallbacks

6. **Layout conventions** — max-width `1480px`, grid gutters (`10px` for stats/controls, `16px` for layouts), header sticky behavior
