# 013: Auto-update support for packaged app

**Status:** inbox
**Created:** 2026-07-20

---

## Problem

The app ships as a Wails-built native binary (`mochila-archive-viewer.app` on macOS). There is no update mechanism: once a user installs a build, they never receive fixes or new features unless they manually rebuild/reinstall. As the app matures (multi-user, multi-platform providers), stale installs will diverge from documented behavior.

## Context / Direct evidence

- Wails v2 has **no built-in auto-updater** (unlike Electron's `autoUpdater`). Common approaches:
  1. **Check-only**: on startup, fetch a version manifest (e.g. GitHub Releases API), compare to the embedded build version, show a "new version available" banner with a download link. Simple; no code-signing complexity or privileged writes.
  2. **Self-replace**: download the new binary and swap it in place (e.g. `minio/selfupdate`). On macOS, replacing files inside a signed .app bundle invalidates the signature — the whole bundle must be replaced and re-verified.
  3. **Sparkle / platform-native**: macOS-native updater; heavier integration.
- The repo has no release pipeline yet (no CI publishing tagged builds), so any updater needs a release channel first.
- Dev builds currently self-sign and warn about private APIs (`wails dev` output, 2026-07-20) — the signing story is unresolved, which gates in-place updating.

## Proposed slices (compoundable)

1. **Embed a build version**: `-ldflags "-X main.Version=..."`, expose via a Wails binding, show in Settings.
2. **Release pipeline**: CI job that builds and attaches artifacts to a GitHub Release on tag push.
3. **Update check**: on startup (and via a Settings button), query the latest release, compare semver, surface a non-blocking "Update available" notice with a download link. Must fail silently offline — the app is a local archive viewer and must work fully offline.
4. *(Deferred)* In-place self-update once the app is properly code-signed/notarized.

## Recommendation

Do slices 1–3 first; slice 4 is gated on code signing.
