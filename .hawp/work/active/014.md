## Problem

The login → dashboard → explorer flow was reworked (login-first entry, logout reset, infinite scroll), but had only been verified by `go test` + `npm run build`. Real UI behavior (login validation, dashboard stats, platform entry, logout returning to a neutral landing state) needs an interactive pass against the running app.

## Approach

- Run `wails dev` from `src/` — Wails serves the app with live Go bindings at `http://localhost:34115`, which a browser can drive directly (bindings bridge over the dev server, no native window needed for smoke testing).
- Drive the UI: login as a user, verify dashboard stats render, enter a platform, scroll the gallery, log out, and confirm the app returns to the login screen with no platform state leaking.
- Record findings as evidence in `.hawp/work/evidence/YYYY/MM/DD/`.

## Acceptance

- [x] App boots via `wails dev` without errors
- [x] Login screen is the entry point (no auto-entry into Snapchat)
- [x] Dashboard shows per-platform stats after login
- [x] Gallery pagination loads additional batches (verified via Load-more; IntersectionObserver auto-trigger could not fire in the embedded test browser due to renderer throttling — wiring reviewed and corrected, verify visually in the native window)
- [x] Logout returns to login screen; re-login does not show stale data
