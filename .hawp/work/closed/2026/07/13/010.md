# 010: Multi-user isolation — wire profile to indexed data

**Status:** done
**Created:** 2026-07-09
**Closed:** 2026-07-13 (implemented in commit eb2c321, "add multi-user isolation with user_id foreign keys on all scoped tables")

---

## Problem

The profile system stores usernames/loggedIn state but indexed data (media, conversations, archives) has **no user_id column**. When User A selects zips and indexes them, User B sees the same data. Profiles are cosmetic.

## Solution: Add user_id foreign key to all scoped tables

### Current Schema

```sql
CREATE TABLE profile (
    profile_id INTEGER PRIMARY KEY CHECK (profile_id = 1),  -- only ONE row!
    username TEXT NOT NULL,
    full_name TEXT NOT NULL,
    logged_in INTEGER NOT NULL
);

CREATE TABLE media_items (platform TEXT, ...);           -- NO user_id
CREATE TABLE conversations (platform TEXT, ...);         -- NO user_id
CREATE TABLE archive_files (platform TEXT, ...);         -- NO user_id
```

### Target Schema

```sql
CREATE TABLE profile (
    profile_id INTEGER PRIMARY KEY AUTOINCREMENT,  -- support multiple users
    username TEXT NOT NULL UNIQUE,
    full_name TEXT NOT NULL DEFAULT '',
    logged_in INTEGER NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE media_items (
    platform TEXT NOT NULL,
    user_id INTEGER NOT NULL REFERENCES profile(profile_id),
    media_id INTEGER NOT NULL,
    ...
    PRIMARY KEY(platform, user_id, media_id)
);

-- Same for: conversations, messages, archive_files, json_files, platform_snapshots
```

## Migration Strategy (backwards compatible)

1. Add `user_id INTEGER NOT NULL DEFAULT 1` to all tables (NOT NULL with default preserves existing data)
2. Update all INSERT/SELECT statements to include user_id filtering
3. Service layer receives activeUser, passes it to store methods
4. Wails bindings extract current session from runtime context

## Files Changed

- `src/internal/archive/storage.go` — schema migrations + all CRUD methods add user_id
- `src/internal/archive/service.go` — pass activeUser through method chain
- `src/appshell/app.go` — Wails bindings inject user context
- `src/frontend/src/App.svelte` — platform selector UI, user-aware loading

## Risks

- Existing databases need migration (handled by ADD COLUMN + DEFAULT)
- All queries change scope (platform → platform+user_id)
- Performance: adding index on user_id column
