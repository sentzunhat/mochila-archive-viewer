# 024 — Media HTTP Serving: Fix `/media/` Routing in Dev Mode

**Status:** in-progress  
**Created:** 2026-07-22  
**Root Cause:** Vite dev server intercepts `/media/` requests before Wails backend ServeHTTP handler

---

## Problem Analysis

### Symptoms
- Media tiles render as white/blank (CSS working, dimensions correct)
- HTTP requests to `/media/snapchat/2/0` return 200 but with HTML body (Vite index)
- Production build would work (no Vite routing), but dev mode is blocked

### Root Cause
In `wails dev` mode:
1. Vite dev server runs on port 5173 internally
2. Wails bridges frontend (Vite) + backend (Go) on localhost:34115
3. Browser requests `/media/snapchat/2/0`
4. Vite intercepts request (doesn't know about `/media/` route)
5. Returns HTML index instead of delegating to backend ServeHTTP

### Verification
- curl to HTTP endpoint returns Vite's `index.html`
- Content-Type is `text/html`
- Size is 344 bytes (HTML doc, not image)
- Backend ServeHTTP handler never called

---

## Solution 1: Vite Middleware (Recommended for Dev)

**Approach:** Add Vite middleware to detect `/media/` requests and skip frontend handling.

**Implementation:**

```typescript
// vite.config.ts
import { defineConfig } from 'vite'
import { svelte } from '@sveltejs/vite-plugin-svelte'

export default defineConfig({
  plugins: [svelte()],
  server: {
    middlewares: [
      // Middleware that runs BEFORE Vite's default SPA middleware
      {
        apply: 'serve',
        enforce: 'pre',
        handle(req, res, next) {
          // Let /media requests pass through without Vite handling
          if (req.url?.startsWith('/media/')) {
            // Don't write anything; let Wails backend handle it
            return next()
          }
          // For other routes, continue with Vite's SPA behavior
          next()
        }
      }
    ]
  }
})
```

**Why This Works:**
- Middleware runs before Vite's SPA fallback (which returns index.html for unknown routes)
- Requests pass through to Wails backend ServeHTTP handler
- No proxy loop (unlike reverse proxy approach)
- Clean separation: Vite handles frontend, backend handles `/media/`

**Pros:**
- ✅ No proxy overhead
- ✅ Preserves Wails architecture
- ✅ Simple, minimal code
- ✅ Works in dev mode

**Cons:**
- ⚠️ Requires Vite-specific configuration
- ⚠️ Middleware API might change in future Vite versions
