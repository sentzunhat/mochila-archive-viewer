## Context

The Instagram provider skeleton exists and is more complete than it looks:

- `src/internal/providers/instagram/indexer.go` (141 lines) — walks ZIP entries, collects media items, finds `message_*.json` files under `*/inbox/` paths, parses them inline.
- `src/internal/providers/instagram/parser.go` (103 lines) — parses `instagramThread` JSON into `types.Conversation` / `types.ChatMessage`.
- `src/internal/providers/instagram/provider.go` — `Status()` already returns `"active"`.

The provider is **not wired into the service**: `service.platform()` returns `ErrPlatformNotSupported` for `id != "snapchat"`. Nothing in `IndexArchives`, `GetMedia`, `GetConversations`, or `MediaBytesForUser` is Instagram-aware yet.

Real export data is available to test against (user has a mid-download Instagram archive).

## Known Gaps in the Existing Code

### G1 — Timestamps not converted from `timestamp_ms`
`parser.go:77`: `lastTime` (epoch ms) captured but `LastCreated` is set to `"unknown"`. `ChatMessage.Created` is never set from `m.TimestampMs`.

**Fix**: `time.UnixMilli(m.TimestampMs).UTC().Format(time.RFC3339)` or `"2006-01-02T15:04:05Z"`.

### G2 — `IsSender` is always `false`
Instagram export includes the account owner's username in `your_instagram_activity/personal_information.json` (or derivable from the thread's `participants` list by cross-referencing the profile). `IsSender` needs to be set by comparing `SenderName` against the logged-in username.

**Fix**: Thread parser needs the owner username at parse time. Pass it from the indexer, which can read `personal_information.json` before parsing threads.

### G3 — Media items not linked to chat messages
`parser.go:69`: `hasAttachment` is detected but `msg.MediaIDs` is only set for share links. Media items in `m.Media[*].image.uri` / `.video.uri` reference ZIP entry paths — these should be resolved to `MediaItem.ID` values so the chat UI can render inline media (same as Snapchat).

**Fix**: Build a uri→MediaItem map in `IndexZips` (keyed by entry path) and use it when parsing threads. Set `msg.MediaIDs` to the resolved `MediaItem.ID` token.

### G4 — `Conversation.ID` uses filename, not a stable unique ID
`parser.go:93`: `ID: filepath.Base(entry)` gives `"message_1.json"`, `"message_2.json"` etc. These collide across threads and don't survive a re-index if Meta changes numbering.

**Fix**: Hash the ZIP name + entry path, or use the thread's `thread_path` field from the JSON (e.g. `"inbox_123456789"`).

### G5 — `Service.platform()` blocks instagram
`service.go`: `if id != "snapchat" { return nil, ErrPlatformNotSupported }`.

**Fix**: Add instagram branch: call `instagram.IndexZips` in `IndexArchives` when `platform == "instagram"`, and lift the `ErrPlatformNotSupported` guard.

## Work Plan

### Step 1 — Inspect real export (inbox)
Open a sample Instagram ZIP and confirm:
- Top-level folder layout (is it `your_instagram_activity/` or `messages/inbox/`?)
- Thread JSON key names (`sender_name`, `timestamp_ms`, etc. — confirm against spec in 009)
- Profile JSON path for owner username
- Media URI format (relative to ZIP root or absolute?)

### Step 2 — Fix gaps G1–G4 in parser/indexer
- Convert `timestamp_ms` → RFC 3339 string.
- Read owner username from profile JSON; pass to thread parser for `IsSender`.
- Resolve `m.Media[*].uri` entries to `MediaItem.ID`; populate `msg.MediaIDs`.
- Stable conversation IDs from `thread_path` or a hash.

### Step 3 — Wire into service
- In `service.go:IndexArchives`, dispatch to `instagram.IndexZips` when `platform == "instagram"`.
- Call `instagram.ParseChatHistory` equivalent (or expose `IndexZips.Conversations()`) to get conversations for `SaveSnapshot`.
- Lift the `ErrPlatformNotSupported` guard for instagram.
- Update `service.go:ZipPath` to handle `ps.Selected` for instagram platform state.

### Step 4 — Serve media
`app.go:ServeHTTP` + `service.go:MediaBytesForUser` use `snapchat.ReadEntry`. Since `instagram.ReadEntry` is identical in implementation, either:
- Extract `ReadEntry` to a shared `providers/common` package and use from both.
- Or call `instagram.ReadEntry` when platform == "instagram" in `MediaBytesForUser`.

### Step 5 — End-to-end smoke test
- Load real Instagram ZIP, click Index, verify media gallery shows photos/videos.
- Open a DM thread, verify messages render with timestamps and inline media.
- Verify `IsSender` correctly highlights outgoing messages.
