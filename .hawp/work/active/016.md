# 016: Snapchat UI parity + polish vs original POC (before Instagram/Facebook UI)

**Status:** inbox
**Created:** 2026-07-20

---

## Problem

Before building Instagram/Facebook explorer UIs, the Snapchat explorer must be brought to (and past) the quality of the original POC (`archive/snapchat-export/app/src/App.svelte` + `styles.css`). The Go/Wails port carried most of the POC over, but an audit on 2026-07-20 found concrete drift.

## Direct evidence (audit 2026-07-20)

1. **Data Explorer view is broken/missing.** The tab exists and sets `view = "data"`, but the current `App.svelte` has no `{:else if view === "data"}` branch (only gallery/today/messages/structure) — clicking the tab renders an empty main area. The POC had a full two-pane JSON explorer (`.data-layout`, `.json-list`, `.json-panel`, `.json-heading`, `.child-grid` with key/type/record counts and pretty JSON sample). Current `style.css` also dropped `.data-layout`, `.json-panel`, `.json-heading` (only `.json-list button` survives).
2. **POC classes vs current**: current added `.structure-*`, `.profile-*`, `.placeholder`, `.secondary-button` (fine), POC-only classes are exactly the Data Explorer set above.
3. Baseline for comparison: POC is 449 lines / 574 CSS lines; current is ~1,380 lines mixing login/dashboard/explorer.

## Scope

1. **Restore the Data Explorer view** from the POC design: json file list aside, heading with entry/zip/top-level/type, child-key grid with record counts, pretty JSON sample panel. Backend bindings (`GetJSONPreview`, `jsonFiles`) already exist and are loaded — this is markup+CSS work.
2. **Side-by-side parity pass** on the remaining views (gallery, on-this-day, messages, structure, media modal): compare against the POC running or its source, and note/fix any lost details (year bars, result-line, message sent/received styling, modal meta list).
3. **Improve as we go** (post-parity):
   - Convert restored/touched sections to Tailwind utilities per the item 012 direction (preflight is off; utilities coexist with legacy classes).
   - Keyboard/a11y: focus states on tiles, Esc to close modals (verify), tab order in the two-pane layouts.
   - Empty/loading states for each view consistent with the new theme.
4. **Exit criteria**: every tab renders a designed view; no dead tabs; visual spot-check in the browser at 34115 with screenshots saved to `.hawp/work/evidence/`.

## Non-goals

- Instagram/Facebook explorer UIs (blocked on this item; they inherit the polished layout).
- Splitting App.svelte into components (worth doing but its own item).
