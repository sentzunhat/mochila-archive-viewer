## Executed (2026-07-21)

1. Added `--surface-dark` (`#171717`) and `--surface-dark-ink` (`#f6f0d5`) tokens to `:root`; replaced all 4 remaining hardcoded hex values in `style.css` (`.preview` background, `.placeholder` text, `.modal-media` background ŌĆö all `--surface-dark`/`--surface-dark-ink` ŌĆö and the `.today-hero` gradient, which reuses existing `--panel`/`--accent-soft`).
2. `.today-hero`'s radial highlight now reads `color-mix(in srgb, var(--accent) 65%, transparent)` instead of a hardcoded Snapchat-yellow `rgba(255, 252, 0, 0.65)` ŌĆö it will follow the active platform's accent once Instagram/Facebook explorers exist, matching the header strip/active tab/sent-message bubble that already do this.
3. Added `--font-sans` token (Inter stack, copied verbatim ŌĆö no visual change) and pointed `body` at it instead of a hardcoded font-family declaration.
4. **Open decision left as-is, not resolved**: keep Inter vs. add a webfont. Recommendation in the implementation plan stands (keep Inter ŌĆö no network dependency, consistent with the app's offline-first architecture); revisit only if there's a specific reason to change fonts.

Verified: `grep` confirms zero hardcoded hex values remain in `style.css` outside the `:root` token definitions themselves. `go build`, `npm run build` pass. Verified live in the browser: media tile/modal letterboxing renders identically (`rgb(23,23,23)` via computed style, matching the original `#171717`), On This Day hero gradient still renders correctly on Snapchat.
