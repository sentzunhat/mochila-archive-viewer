## Problem

Before building Instagram/Facebook explorer UIs, the Snapchat explorer must be brought to (and past) the quality of the original POC (`archive/snapchat-export/app/src/App.svelte` + `styles.css`). The Go/Wails port carried most of the POC over; this item tracks parity + polish work found by comparing the two live.

## Correction (2026-07-20)

The original write-up claimed the Data Explorer tab had no rendering branch. That was **wrong** — verified live in the browser: Data Explorer works correctly (file list, heading, top-level keys, snapshot path, pretty JSON all render). The code reuses `.messages-layout`/`.message-panel`/`.structure-grid` styling via the trailing `{:else}` branch rather than dedicated `.data-layout`/`.json-panel` classes from the POC, which is a naming/cleanliness nit, not a functional gap. Leaving this note so future passes don't re-investigate a non-issue.

## Real bugs found and fixed (2026-07-20, commit 31f4078)

1. **Gallery search/year/category/type filters were dead.** Once a platform is selected (the only reachable flow), the gallery always rendered `paginatedMedia`, but search/category/type only touched an unused client-side filter chain, and the year picker refetched an array nothing displayed. Fixed by threading a `MediaFilter` (year/category/type/search) through storage → service → appshell → frontend; `GetMediaPaginated`/`GetMediaCount` are now the single source of truth. Filter changes reset and reload the paginated query, guarded by a request-generation counter so a slow superseded request can't clobber a newer one.
2. **Messages tab showed a conversation heading but zero message bubbles** until manually clicked. Root cause #1: Go serializes a nil slice as `null` not `[]`, so the auto-load guard's `.length === 0` check never matched `null`. Root cause #2 (only surfaced after fixing #1): the "already loaded" id-tracking `Set` wasn't cleared when `loadPlatform()` re-fetched a fresh snapshot, so the pre-login `onMount` auto-load would mark a conversation loaded, then the post-login `selectPlatform → loadPlatform` call would silently wipe its messages back to null with the guard now permanently blocking the retry. Both fixed; verified live (first conversation's 9 messages render with no click).

## More bugs found and fixed (2026-07-20, side-by-side parity pass)

3. **Clicking anywhere inside the media modal closed it.** The POC's inner modal `<div>` had `on:click={keepModalOpen}` (stops propagation) so only a true backdrop click closes the modal; the Go port dropped this guard while keeping the backdrop's close-on-click handler, so clicking the metadata panel, the image, anywhere inside, bubbled up and closed the modal. The other two modals (settings, profile) already had `on:click|stopPropagation` — only the media modal was missing it. Fixed; verified live (click inside stays open, backdrop click and Escape both still close it).
4. **Chat bubbles showed the literal type string ("TEXT") as if it were the message.** Not a bug in parsing — confirmed against the live archive that ~29% of messages genuinely have empty `content` (Snapchat's export doesn't retain text for many ephemeral/expired chat types). But rendering the raw `mediaType` enum value as message text read as if the user had typed the word "TEXT". Replaced with a small label map ("No text saved", "Shared story", "Location share", etc.) rendered in muted italics, visually distinct from real content.

## Scope (remaining)

1. **Tailwind conversion pass** on the remaining hand-styled sections per item 012 direction (preflight is off; utilities coexist with legacy classes) — lower priority now that the major interaction bugs are fixed.
2. **Filter-change performance**: switching a filter while up to 180 concurrent `GetMediaSource` calls are still resolving from the previous filter makes the UI feel sluggish for a few seconds (see item 017 evidence). Consider cancelling in-flight `ensureMediaSources` work on filter change, or capping concurrency.
3. **Exit criteria**: every tab renders a designed view (✓); no dead tabs (✓); core interactions (search/filter/pagination/messages/modal) verified live, not just visually (✓); screenshots/evidence saved to `.hawp/work/evidence/` (✓ see `2026/07/20/016-gallery-filter-and-messages-fix.md`). Remaining scope above is polish, not blocking — safe to start Instagram/Facebook UI work in parallel if desired.
