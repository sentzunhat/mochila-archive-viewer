## Facebook Export Format (from 009 research + Meta's current download spec)

Facebook's "Download Your Information" export ZIP contains:

```
messages/
  inbox/
    <thread_name>_<id>/
      message_1.json        ← main thread file (may split into message_2.json etc.)
      photos/               ← media attached to this thread
      videos/
  archived_threads/         ← same structure, archived DMs
  filtered_requests/        ← message requests
your_facebook_information/
  profile_information/
    profile_information.json  ← { "profile_v2": { "name": { "full_name": "..." } } }
  friends_and_followers/
    friends_of_friends.json
```

### Thread JSON structure (`message_1.json`)
```json
{
  "participants": [{ "name": "Alice Smith" }, { "name": "Bob Jones" }],
  "messages": [
    {
      "sender_name": "Alice Smith",
      "timestamp_ms": 1620000000000,
      "content": "hey",
      "reactions": [{ "reaction": "❤️", "actor": "Bob Jones" }],
      "photos": [{ "uri": "messages/inbox/alice_123/photos/img.jpg", "creation_timestamp": 1620000001 }],
      "videos": [{ "uri": "messages/inbox/alice_123/videos/vid.mp4", "creation_timestamp": 1620000002 }],
      "audio_files": [{ "uri": "..." }],
      "sticker": { "uri": "messages/stickers_used/..." }
    }
  ],
  "title": "Alice Smith",
  "is_still_participant": true,
  "thread_path": "inbox/alice_123",
  "thread_type": "Regular"
}
```

**Key differences from Instagram** (same base shape, but):
- `photos` / `videos` arrays instead of `media[*].image` / `media[*].video`
- Photo URI is relative to ZIP root (not relative to thread folder)
- Thread ID is `thread_path` (e.g. `"inbox/alice_123456"`)
- Older exports may use `.js` files with `window.__FB_EXPORT_DATA` wrapper — needs stripping
- Text content may be Mojibake (Latin-1 escaped as UTF-8 bytes) — common in older exports, requires `strings.Map` recode

## Work Plan

### Step 1 — Inspect real export
Open the Facebook ZIP and confirm:
- Exact folder layout (does it match spec above, or is it a newer format?)
- Whether `.js` Mojibake issues are present in this export
- Whether `thread_path` is reliably unique
- Which media URI format is used (relative to ZIP root vs relative to thread?)

### Step 2 — Create `facebook/indexer.go`
Model on `instagram/indexer.go`:
```go
func IndexZips(paths []string) (*Index, error)
func ReadEntry(zipPath, entry string) ([]byte, error)
func ReadEntryString(zipPath, entry string) (string, error)
```

Walk ZIP entries:
- Collect media items from `messages/inbox/*/photos/` and `messages/inbox/*/videos/`
- Collect JSON thread files matching `messages/inbox/*/message_*.json`
- Parse owner username from `your_facebook_information/profile_information/profile_information.json`
- Call thread parser for each message file; accumulate conversations

### Step 3 — Create `facebook/parser.go`
```go
func parseThread(zipPath, entry, ownerName string) *types.Conversation
```

- Unmarshal thread JSON (strip `window.__FB_EXPORT_DATA = ` prefix if present)
- Convert `timestamp_ms` → RFC 3339 string (same helper as Instagram)
- Set `IsSender` by comparing `sender_name` with `ownerName`
- Resolve `photos[*].uri` / `videos[*].uri` to `MediaItem.ID` via entry→ID map
- Set `msg.MediaIDs` to token for HTTP media resolution
- Stable conversation ID from `thread_path` (already unique)

### Step 4 — Update `facebook/provider.go`
Change `Status()` from `"planned"` to `"active"`.

### Step 5 — Wire into service
Same as Instagram step 3: dispatch `facebook.IndexZips` from `service.IndexArchives` when `platform == "facebook"`. Lift `ErrPlatformNotSupported` guard.

### Step 6 — Shared `ReadEntry` helper (if not done in 021)
If 021 extracts `ReadEntry` to `providers/common`, use it here too. If not, implement directly as in the Instagram copy.

### Step 7 — End-to-end smoke test
- Load real Facebook ZIP, click Index, verify media gallery shows photos/videos.
- Open a Messenger thread, verify messages render with timestamps and inline photos.
- Verify `IsSender` highlights outgoing messages correctly.
- Verify Mojibake is absent (or add recode if needed after inspecting real data).
