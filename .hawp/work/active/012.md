## Problem (original framing, 2026-07-13 — see design docs for current state)

`style.css` had hardcoded hex values that should reference tokens, no centralized design guide, and Tailwind config drift. The two linked docs above are the actual deliverables and now reflect reality as of 2026-07-21 — read those, not this historical section, for current color values, token names, spacing scale, and the theming mechanism.

## Remaining scope (see implementation plan for exact steps)

1. Replace 4 hardcoded hex values in `style.css` with 2 new tokens (`--surface-dark`, `--surface-dark-ink`) plus reuse of existing `--panel`/`--accent-soft` — exact lines and rationale in the implementation plan.
2. Make `.today-hero`'s gradient follow the active platform accent, matching the rest of the UI (found during the 2026-07-21 audit, not in the original scope).
3. Optional: formalize `--font-sans` as a token (currently hardcoded directly on `body`).
4. Open decision, not yet made: keep Inter (no changes, no network dependency) or add a webfont — see implementation plan's recommendation.

## Out of Scope (unchanged)

- Dark mode support
- Redesigning existing layouts (token/color-only pass)

## Risk

Low — token substitutions with identical or near-identical resulting values; see implementation plan's Risk section for the one visual check worth doing (the `.today-hero` gradient becomes platform-reactive, verify on all three platforms, not just Snapchat).
