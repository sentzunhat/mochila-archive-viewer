# 011: Profile Management — proper multi-user creation and switching flow

**Status:** inbox  
**Created:** 2026-07-09

---

## Problem

After #010 added user_id to all tables, the backend supports multiple users but the UI does not:

1. **No profile creation from UI** — SaveProfile only creates/updates one profile; there's no way to add additional users
2. **switchUser loads wrong profile** — It calls GetFrontendState which activates the LOGGED_IN user (the "current session"), not the clicked user
3. **No visual indicator** of which user's data is being viewed
4. **Settings/Profile modal is minimal** — Just edit fields with no create new profile flow

## Solution

### Backend Changes (minimal)

1. Add `SelectUser(userId int64)` to Service — sets activeUserId and logs that this user is now active
2. Wails binding: `func (a *App) SelectUser(id int64) (*archive.Profile, error)` — sets active user without changing loggedIn flag

### Frontend Changes

1. **Profile Modal** — Add "Create New Profile" button that clears fields and allows creation of a new username
2. **AvailableUsers list** — Each user has a "Select" button that calls SelectUser(userId) instead of SaveProfile
3. **Header indicator** — Show current username next to profile name
4. **Fix switchUser** — Use the new Wails binding that sets activeUserId without requiring loggedIn=true

## Files Changed

- `src/internal/archive/service.go` — Add SelectUser method
- `src/appshell/app.go` — Add SelectUser Wails binding
- `src/frontend/src/App.svelte` — Profile modal improvements, user list with select buttons, current user indicator

## Risks

- None major — pure UI/UX improvement, no schema changes
