# 005: Frontend - Refine search UX with keyboard shortcuts

**Status:** inbox  
**Created:** 2025-07-09  
**Related Commit:** c5af4df

---

## Context

Search was added to the gallery in commit c5af4df, but lacks keyboard shortcuts and focus management. Users currently can't quickly jump to search with `/` or clear with Escape — both common patterns in desktop apps.

## Scope

- Add `/` keybinding to focus search input
- Add `Escape` keybinding to clear search query and blur
- Ensure search input retains focus after typing (no re-render loss)
- Optional: add Cmd/Ctrl+F to focus search from any view

## Files

- `src/frontend/src/App.svelte` — keyboard handlers, search input ref

## Risks

- Svelte 5 runes store bindings may interfere with keyboard event handling
- Focus management across view changes (gallery → today → messages)

## Verification

- `/` focuses search from gallery view
- `Escape` clears query and blurs without losing filtered state on re-enter
- Cmd/Ctrl+F works when implemented
