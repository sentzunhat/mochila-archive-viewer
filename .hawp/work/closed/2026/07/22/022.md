## Export Format — Verified Against Real Data (2026-07)

Real Facebook "Download Your Information" export structure:

```
your_facebook_activity/
  messages/
    inbox/
      <thread_id>/
        message_1.json              ← thread data (split into message_2.json, message_3.json)
        photos/
        videos/
        gifs/
        audio/
    archived_threads/               ← same structure
    e2ee_cutover/                   ← encrypted conversations
    stickers_used/
  posts/
    media/stickers_used/
personal_information/
  profile_information/
    profile_information.json        ← { "profile_v2": { "name": { "full_name": "..." } } }
```

### Message JSON structure
```json
{
  "participants": [
    { "name": "Alice Smith" },
    { "name": "Bob Jones" }
  ],
  "messages": [
    {
      "sender_name": "Alice Smith",
      "timestamp_ms": 1620000000000,
      "content": "hello",
      "is_geoblocked_for_viewer": false,
      "is_unsent_image_by_messenger_kid_parent": false,
      "reactions": [{ "reaction": "❤", "actor": "Bob Jones" }],
      "photos": [
        {
          "uri": "your_facebook_activity/messages/inbox/thread_id/photos/123.jpg",
          "creation_timestamp": 1620000001
        }
      ]
    }
  ]
}
```

### Profile JSON structure
```json
{
  "profile_v2": {
    "name": {
      "full_name": "Alice Smith",
      "first_name": "Alice",
      "last_name": "Smith"
    }
  }
}
```

**Key findings:**
- URIs are absolute paths from ZIP root (start with `your_facebook_activity/`)
- Messages split across multiple files per thread (`message_1.json`, `message_2.json`, etc.)
- Thread ID is directory folder name (e.g. `thefamjam_4313518522014579`)
- Support: photos, videos, gifs, audio, reactions
- Text content has proper UTF-8 (no Mojibake in current exports)
- Newer format uses `personal_information/` not `your_facebook_information/`

## Work Plan

### Step 1 — Create `facebook/parser.go`

```go
type Message struct {
  SenderName     string
  TimestampMs    int64
  Content        string
  Photos         []MediaRef
  Videos         []MediaRef
  Gifs           []MediaRef
  Audio          []MediaRef
  Reactions      []Reaction
  // Flags omitted for now (is_geoblocked, etc.)
}

type MediaRef struct {
  URI               string
  CreationTimestamp int64
}

type Reaction struct {
  Reaction string // emoji
  Actor    string // name
}

func parseThreadFile(zipPath, entry string, ownerName string, media map[string]int) (*types.Conversation, error)
  - Unmarshal JSON (standard JSON, no wrapper stripping needed)
  - Extract all participants
  - Convert messages: timestamp_ms → RFC3339, resolve media URIs to IDs
  - Set IsSender by comparing sender_name against ownerName
  - Return Conversation
```

### Step 2 — Create `facebook/indexer.go`

```go
func IndexZips(paths []string) (*Index, error)
  - Walk each ZIP
  - Collect media: `your_facebook_activity/messages/inbox/*/photos/`, `*/videos/`, `*/gifs/`, `*/audio/`
  - Collect message files: `your_facebook_activity/messages/inbox/*/message_*.json`
  - Parse profile from `personal_information/profile_information/profile_information.json`
  - For each thread directory: call parseThreadFile for each message_*.json
  - Accumulate into types.Index

func ReadEntry(zipPath, entry string) ([]byte, error)
  - Fetch raw bytes from ZIP
  
func ReadEntryString(zipPath, entry string) (string, error)
  - Fetch as string
```

### Step 3 — Update `facebook/provider.go`

Change `Status()` from `"planned"` to `"active"`.

### Step 4 — Wire into service
