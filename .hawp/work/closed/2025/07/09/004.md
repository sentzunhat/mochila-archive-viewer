## Context

The backend storage layer was rebuilt for multi-user support (users, platforms, user_platforms tables), but the Wails bindings in `appshell/app.go` and Service methods still use the old single-user API. The `Profile` type in frontend doesn't match what's returned from the backend. This means the profile system exists in theory but isn't wired correctly through the Wails RPC layer.

## Scope

- Thread active user UUID through `GetFrontendState()` return
- Update `LoadProfile()`/`SaveProfile()`/`LogoutProfile()` to use multi-user Store methods
- Align frontend Profile type with backend UserEntry structure
- Add `AvailableUsers()` method for user switching (deferred if scope grows)

## Files

- `src/appshell/app.go` — Wails bindings
- `src/internal/archive/service.go` — Service layer calls
- `src/frontend/src/App.svelte` — Frontend Profile type alignment

## Risks

- Breaking existing frontend state shape if Profile struct changes
- Migration edge case: legacy users may not exist in new schema
