# 025 — Date Extraction for Instagram & Facebook Media

**Status:** in-progress  
**Created:** 2026-07-22  
**Depends on:** 021 (Instagram), 022 (Facebook) — providers complete and indexed

---

## Problem

Currently:
- **Snapchat:** Dates extracted from filenames (e.g., `2022-07-15_photo.jpg`) ✅
- **Instagram:** Uses same regex pattern but many files lack dates in paths → defaults to "unknown"
- **Facebook:** Hard-coded to `idx.Years["unknown"]` → ALL media marked as "unknown" ❌

**Impact:** Users cannot filter media by year for Instagram/Facebook; gallery shows "unknown" category instead of browseable year range.

---

## Export Format Research

### Snapchat Structure
```
memories/
  2022/
    2022-07-15_abcdef.jpg       ← date in filename
    2022-07-15_xyz.mp4
  2023/
    2023-01-20_photo.jpg
```
**Date Source:** Filename contains `YYYY-MM-DD` prefix.

---

### Instagram Export Structure

Based on common "Download Your Information" exports:

```
your_instagram_activity/
  content/
    photos/
      year_2022/
        2022_07_15_abc.jpg       ← date in path or filename
      year_2023/
  posts/
    posts_1.json                 ← metadata JSON
      [
        {
          "creation_timestamp": 1660521600,
          "media": [
            {
              "uri": "path/to/file.jpg",
              "title": "photo"
            }
          ]
        }
      ]
```

**Potential Date Sources:**
1. **Filename pattern:** Look for `YYYY_MM_DD` in filenames (similar to current regex)
2. **JSON metadata:** Parse `posts_*.json` files that contain `creation_timestamp` (Unix epoch in seconds)
3. **Directory structure:** Some exports organize by `year_YYYY/` directory

**Key Insight:** Instagram might store creation times in JSON metadata files separate from media files.

---

### Facebook Export Structure

```
your_facebook_activity/
  messages/
    inbox/
      thread_name_id/
        message_1.json           ← thread metadata + message data
          {
            "messages": [
              {
                "timestamp_ms": 1660521600000,  ← milliseconds epoch!
                "content": "text",
                "photos": [                     ← media references
                  {
                    "uri": "path/to/photo.jpg",
                    "creation_timestamp": 1660521600
                  }
                ]
              }
            ]
          }
  photos/
    year_2022/                   ← organized by year
    year_2023/
```

**Date Sources (in priority order):**
1. **Message references:** Media in `messages/inbox/*/message_N.json` has `creation_timestamp`
2. **Directory structure:** Photos organized as `photos/year_YYYY/` 
3. **Extracted from message timestamp:** If media has no date, use message `timestamp_ms`

## Implementation Strategy

### Phase 1: Instagram Date Extraction

**Approach:** Parse `posts_*.json` and link timestamps to media files.

```go
// instagram/indexer.go additions

func indexZip(zipI int, zipPath string, idx *Index, mediaByURI map[string]int) (ZipMeta, error) {
  // ... existing code ...
  
  // NEW: Extract dates from JSON metadata
  postDates := extractPostDates(reader)  // map[mediaURI]timestamp
  
  // When collecting media files:
  for _, f := range reader.File {
    if isMediaFile(f.Name) {
      entry := f.Name
      id := len(idx.Media)
      
      // Try to find date from JSON metadata
      date := "unknown"
      if timestamp, found := postDates[normalizeURI(entry)]; found {
        date = formatTimestamp(timestamp)  // Unix epoch → YYYY-MM-DD
      } else if dateFromName := extractDate(entry); dateFromName != "unknown" {
        date = dateFromName
      }
      
      year := "unknown"
      if date != "unknown" {
        year = date[:4]
      }
      
      // ... rest of media item creation ...
    }
  }
}

func extractPostDates(reader *zip.ReadCloser) map[string]int64 {
  dates := make(map[string]int64)
  
  for _, f := range reader.File {
    if strings.Contains(f.Name, "posts") && strings.HasSuffix(f.Name, ".json") {
      data, _ := readZipFile(f)
      var posts []struct {
        CreationTimestamp int64 `json:"creation_timestamp"`
        Media []struct {
          URI string `json:"uri"`
        } `json:"media"`
      }
      
      if json.Unmarshal(data, &posts) == nil {
        for _, post := range posts {
          for _, media := range post.Media {
            dates[normalizeURI(media.URI)] = post.CreationTimestamp
          }
        }
      }
    }
  }
  
  return dates
}

func formatTimestamp(epochSeconds int64) string {
  t := time.Unix(epochSeconds, 0)
  return t.Format("2006-01-02")
}
```

### Phase 2: Facebook Date Extraction

**Approach:** Extract dates from message metadata and directory structure.

```go
// facebook/indexer.go modifications

func indexZip(zipI int, zipPath string, ...) (ZipMeta, error) {
  // ... existing code ...
  
  // Parse year from directory structure: photos/year_YYYY/
  yearDirMap := extractYearDirs(reader)  // map[entry]year
  
  // When collecting media:
  for _, f := range reader.File {
    if isMediaFile(f.Name) {
      entry := f.Name
      id := len(idx.Media)
      
      // Priority 1: Extract from directory structure (photos/year_2022/)
      if year, found := yearDirMap[entry]; found {
        idx.Years[year]++
        item := types.MediaItem{
          ID: id,
          Year: year,
          Date: year + "-01-01",  // Fallback to Jan 1 of that year
          // ... rest of item ...
        }
        idx.Media = append(idx.Media, item)
        continue
      }
      
      // Priority 2: Will be linked from messages later (see Phase 2b)
      idx.Years["unknown"]++
      // ... create item with unknown date ...
    }
  }
  
  // Phase 2b: Link media to message timestamps
  for threadID, messageFiles := range threads {
    for _, msgEntry := range messageFiles {
      data, _ := readZipFile(reader, msgEntry)
      var thread struct {
        Messages []struct {
          TimestampMs int64 `json:"timestamp_ms"`
          Photos []struct {
            URI string `json:"uri"`
          } `json:"photos"`
          // ... videos, gifs, audio ...
        } `json:"messages"`
      }
      
      json.Unmarshal(data, &thread)
      
      // For each media reference, update its date if "unknown"
      for _, msg := range thread.Messages {
        timestamp := msg.TimestampMs / 1000  // Convert ms to seconds
        date := formatTimestamp(timestamp)
        year := date[:4]
        
        for _, photo := range msg.Photos {
          mediaURI := normalizeURI(photo.URI)
          if mediaID, found := mediaByURI[mediaURI]; found {
            // Update the media item's date
            idx.Media[mediaID].Date = date
            idx.Media[mediaID].Year = year
            
            // Update year counts
            if idx.Media[mediaID].Year == "unknown" {
              idx.Years["unknown"]--
            }
            idx.Years[year]++
          }
        }
      }
    }
  }
}

func extractYearDirs(reader *zip.ReadCloser) map[string]string {
  yearMap := make(map[string]string)
  
  for _, f := range reader.File {
    // Match: photos/year_2022/file.jpg
    re := regexp.MustCompile(`photos/year_(\d{4})/`)
    if matches := re.FindStringSubmatch(f.Name); matches != nil {
      yearMap[f.Name] = matches[1]
    }
  }
  
  return yearMap
}
```
