## Problem

Three Go files carry unrelated concerns in one blob; the 1369-line Svelte frontend is all one component. Adding Instagram + Facebook makes this worse — each new provider adds more conditionals to already-large files.

| File | Lines | Problem |
|------|-------|---------|
| `src/internal/archive/storage.go` | 1054 | Migrate, media, conversations, profiles, snapshots, zips — 6 concerns |
| `src/internal/archive/service.go` | 530 | Index, media, conversations, JSON, profiles — 5 concerns |
| `src/appshell/app.go` | 407 | All Wails bindings in one file, ServeHTTP mixed in |
| `src/frontend/src/App.svelte` | 1369 | All UI: login, gallery, messages, data explorer, settings, modals |

## Principles

- Same package (`archive`, `appshell`) — no import cycle risk, purely file splits.
- Each file has one clear responsibility stated in a one-line comment at the top.
- No new abstractions invented — just reorganization of existing code.
- `go build ./...` and `go vet ./...` must pass after every file move.
- Svelte split is the largest slice; do Go side first since it verifies with the compiler.
